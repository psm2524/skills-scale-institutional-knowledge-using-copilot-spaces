# Personas Application Checklist

Purpose: A short checklist teams can use when assigning personas to a project and confirming handoffs.

- [ ] Identify required personas for this initiative (check the roles table in octoacme-roles-and-personas.md)
- [ ] Assign owners and backups for each persona
- [ ] Document primary collaboration points and handoffs in the project README
- [ ] Confirm acceptance criteria ownership (who verifies success metrics)
- [ ] Confirm security and data instrumentation touchpoints with Security Engineer and Data Analyst
- [ ] Add persona contact and escalation path to the project Risk Register or README
- [ ] Review and update the checklist at the end of planning (sign-off by PM and Product Lead)
